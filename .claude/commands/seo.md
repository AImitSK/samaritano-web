# SEO Audit

Fuehre einen SEO-Audit fuer das Projekt durch.

## Pruefpunkte

### 1. Meta Tags
- [ ] Title auf jeder Seite (50-60 Zeichen)
- [ ] Description auf jeder Seite (150-160 Zeichen)
- [ ] Viewport Meta Tag
- [ ] Charset Meta Tag

### 2. Open Graph
- [ ] og:title
- [ ] og:description
- [ ] og:image (1200x630px empfohlen)
- [ ] og:url
- [ ] og:type

### 3. Twitter Cards
- [ ] twitter:card
- [ ] twitter:title
- [ ] twitter:description
- [ ] twitter:image

### 4. Strukturierte Daten
- [ ] JSON-LD Schema vorhanden
- [ ] Organization Schema
- [ ] WebSite Schema
- [ ] BreadcrumbList (falls Unterseiten)

### 5. Technisch
- [ ] Canonical URLs gesetzt
- [ ] Sitemap vorhanden und aktuell
- [ ] robots.txt korrekt
- [ ] Keine Duplicate Content Issues
- [ ] hreflang (falls mehrsprachig)

### 6. Content
- [ ] H1 auf jeder Seite (nur einmal)
- [ ] Heading-Hierarchie korrekt (H1 > H2 > H3)
- [ ] Alt-Texte fuer alle Bilder
- [ ] Interne Verlinkung

## Ausgabe

Erstelle eine Checkliste mit Status:
- OK - Korrekt implementiert
- FEHLT - Muss hinzugefuegt werden
- WARNUNG - Verbesserung empfohlen

Pruefe: $ARGUMENTS oder alle Seiten
